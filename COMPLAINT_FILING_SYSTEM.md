# Comprehensive Complaint Filing System
## Up-to-date, Multi-Venue, Maximum Impact

### WHAT THIS IS
A system that identifies **ALL possible venues** where users can file complaints, provides **current procedures** for each, and learns **which venues actually get results**.

### WHY THIS MATTERS
When users have issues (discrimination, habitability, illegal eviction, etc.), they often don't know WHERE to file complaints or HOW to file effectively. This system:

1. **Identifies ALL venues** (federal, state, county, city, tenant unions, media)
2. **Provides current procedures** (automatically updated, user-verified)
3. **Shows effectiveness** (based on real outcomes - what actually works)
4. **Multi-venue strategy** (file with multiple places simultaneously for maximum pressure)

### HOW IT WORKS

#### 1. User Describes Issue
- **Issue type**: Discrimination, habitability, Section 8, illegal eviction, etc.
- **Location**: City, county, state, ZIP
- **Situation**: Section 8? Disability? Urgency?

#### 2. System Identifies ALL Applicable Venues

**Federal Venues** (apply everywhere):
- **HUD Fair Housing**: Discrimination complaints (race, disability, familial, sex, religion, national origin)
- **HUD Section 8**: Section 8 voucher discrimination, PHA issues, payment problems
- **ADA (DOJ)**: Disability discrimination, reasonable accommodation denied, service animals
- **FTC**: Consumer fraud, deceptive practices

**State Venues**:
- **Attorney General**: Consumer complaints, fraud, illegal fees, security deposit violations
- **State Housing Agency**: Housing code violations, landlord-tenant disputes
- **State Health Department**: Health hazards, mold, pest infestations

**County Venues**:
- **County Health Department**: Immediate health hazards (no heat, no water, mold, pests)
- **County Court**: Housing court, small claims court

**City Venues**:
- **Code Enforcement**: Building code violations, habitability issues (VERY EFFECTIVE - usually 90%+ success)
- **Rental Licensing Board**: Unlicensed properties, licensing violations

**Other Venues**:
- **Tenant Unions**: Advocacy, organizing, legal support
- **Legal Aid**: Free legal representation
- **Media**: Public pressure (local news, social media)

- **Low-success venues** demoted or flagged for review

### DATA ACCURACY & UPDATES

#### How Procedures Stay Current

1. **Automatic verification** (< 90 days old = current, > 90 days = flag for update)
2. **User validation** (users report if procedures have changed)
3. **Outcome tracking** (users report "this worked" or "this didn't work")
4. **Multi-source verification** (cross-check procedures against official websites, legal aid, user reports)

#### Effectiveness Scoring

**VERIFIED âœ“** (90%+ success, 10+ cases):
- "Eagan Code Enforcement: 95% success rate (12 cases)"

**HIGH** (80%+ success, 5+ cases):
- "HUD Fair Housing: 85% success rate (45 cases)"
- Recommended

**GOOD** (70%+ success, 3+ cases):

**LEARNING** (< 3 cases):
- Shown but marked as "Limited data"

**FLAGGED** (< 60% success, 5+ cases):
- Hidden from users
- Flagged for review (why is this failing?)

### USER EXPERIENCE

#### What Users See

1. **Simple form**:
   - Issue type (dropdown)
   - Location (city, state, ZIP)
   - Situation (Section 8? Disability? Urgency?)
   - Description (optional)

2. **Comprehensive strategy**:
   - **Success probability** (Very High, High, Moderate)
   - **Immediate action** (most effective venue, with contact info)
   - **Simultaneous filings** (other venues to file with today)
   - **Timeline** (what to expect when)
   - **Escalation path** (if first attempts fail)

3. **For each venue**:
   - **Venue name** (e.g., "HUD Fair Housing")
   - **Contact info** (phone, website, online form link)
   - **Success rate** (85% based on 45 cases)
   - **Timeline** (100 days investigation)
   - **Why file here** (federal protection, cannot retaliate)
   - **Forms required** (link to online form or downloadable PDF)
   - **Fees** (Free or $X)

#### What Users Do NOT See

- Internal effectiveness calculations
- Source code
- Database structure
- Failed venues (< 60% success hidden automatically)

#### Location Intelligence
- **Auto-discover local venues**: City code enforcement, county health, etc.

- **Local variations**: Procedures vary by jurisdiction (system learns each)

#### Adaptive Registration
- **Learn from outcomes**: Users report "I filed here, here's what happened"
- **Update procedures**: If user reports procedures changed, system updates
- **Improve recommendations**: High-success venues promoted, low-success demoted

### API ENDPOINTS

#### POST `/api/complaint/identify-venues`
Identify all applicable venues for issue.

**Request**:
```json
{
  "issue_type": "no_heat",
  "location": {
    "city": "Eagan",
    "state": "Minnesota",
    "county": "Dakota County",
    "zip": "55122"
  },
  "user_situation": {
    "has_section8": false,
    "has_disability": false,
    "description": "No heat for 2 weeks in winter"
  },
  "urgency": "emergency"
}
```

**Response**:
```json
{
  "success": true,
  "strategy": {
    "issue_type": "no_heat",
    "urgency": "emergency",
    "success_probability": "Very High (90%+)",
    "immediate_actions": [...],
    "simultaneous_filings": [...],
    "escalation_path": [...],
    "timeline": {...}
  },
  "total_venues": 5,
  "confidence": "Very High (90%+)"
}
```

### DEPLOYMENT CONSIDERATIONS

#### Data Storage
- `data/filing_venues.json` - All known venues with contact info
- `data/filing_procedures.json` - Current procedures per venue/location
- `data/filing_outcomes.json` - Outcome tracking (effectiveness scores)

#### Performance
- **Fast venue lookup** (indexed by jurisdiction + issue type)
- **Async procedure updates** (don't block user if discovering new local resources)
- **Cached effectiveness scores** (recalculated daily, not per request)

#### Privacy
- **User anonymity**: No PII required to get filing strategy
- **Outcome reporting**: Optional, encourages participation but not required
- **Aggregate data only**: Individual cases not identifiable
